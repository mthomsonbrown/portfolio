Saplyn

My newest pet project. I often find myself wanting to track data for some random occurrence in life. I'll search the app store and find activity-specific options that push the user down some path defined by the developer that may or may not line up with my goals. On top of that, as separate applications, there's no implicit synchronization between data sets. Enter Saplyn, a place to record generic data.

Rails This is the current backend, but I'm thinking of changing over to some flavor of a Node stack. There aren't that many features right now beyond authentication, and Node seems like the way to go these days. Android Old This was the first stab. This is basically just an authentication flow talking to the Rails backend, and I will probably get rid of it (merge it?) after I incorporate the functionality into the new application.
